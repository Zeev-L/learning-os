```
היי! זה פרומפט להתקנת "Learning OS" — דשבורד למידה אישי שמרכז תוכן מקצועי
(AI · Product · R&D/Eng-Ops · אסטרטגיה) לסיכומי עברית עם שורה תחתונה, משמעויות
והמלצות פרקטיות, + בייסליין של בסט-פרקטיסז וספריית מקורות.
תצוגה חיה לדוגמה: https://zeev-l.github.io/learning-os/

בצע אוטונומית, אמת כל שלב בפועל (לא "נראה תקין"), ובסוף דווח מה מוכן ומה דורש פעולה ידנית ממני.

## 0. זהה מי אני (חשוב — קובע בעלים מול אורח)
הרץ `gh auth status`; אם אני לא מחובר — הנחה אותי ל-gh auth login (אינטראקטיבי, אני אעשה בעצמי).
ואז `gh api user --jq .login`:
- אם התוצאה היא **Zeev-L** (בעל ה-repo המקורי) → אני הבעלים: שכפל ישירות
  `git clone https://github.com/Zeev-L/learning-os.git`, השאר את התוכן, דחיפות הולכות ל-origin שלי.
- אם התוצאה **שונה** → אני אורח: עשה fork לחשבון שלי ושכפל: `gh repo fork Zeev-L/learning-os --clone`.
  אחרי ההתקנה, הצע לי להתחיל **Digest נקי** (למחוק את התוכן האישי של הבעלים ולהשאיר שלד).

## 1. דרישות
בדוק git, node, gh (GitHub CLI). אם חסר — תן פקודת התקנה (macOS: brew) ועצור עד שאתקין.

## 2. התקנה
היכנס לתיקייה שנוצרה, והרץ `npm install`.

## 3. אימות
`node scripts/serve.mjs` ואז http://localhost:8005 — ודא 4 טאבים
(Digest / בייסליין / מקורות / ארכיון) וכרטיסי Digest נטענים. דווח מה ראית בפועל.

## 4. שימוש
לרענן: בשיחת Claude Code אבקש "רענן את ה-Digest" (מושך RSS + GitHub Trending, מסכם לעברית).
לעדכן את העותק שלי: git add/commit/push — הולך אוטומטית ל-origin שלי (ה-repo/fork שלי, לא של אף אחד אחר).

## אופציונלי (הצע לי)
- אתר ציבורי משלי: GitHub → ה-repo שלי → Settings → Pages → main/root → https://<my-user>.github.io/learning-os/
- התאמה: לערוך data/sources.js (הוסף/הסר מקורות לפי התחומים שלי).
- רענון אוטומטי שבועי: לפי SETUP.md (דורש claude CLI; bash scripts/install-schedule.sh).

## סיכום
דווח טבלה: מה מוכן, ומה דורש פעולה ידנית ממני (gh login / Digest נקי / Pages).

הערה: הקובץ SETUP.md ב-repo מכיל את כל ההוראות המלאות.
```

## איך זה מבחין ביני לבין משתמש אחר
- **בהתקנה:** שלב 0 מריץ `gh api user` — אם זה `Zeev-L` (הבעלים) הוא משכפל ישירות; כל אחד אחר עושה **fork** לחשבון שלו.
- **בריצה:** לכל עותק יש `origin` משלו (ה-repo של מי ששכפל) + `gh auth` משלו. `git push` תמיד הולך ל-origin הנכון — אין ערבוב בין משתמשים.
