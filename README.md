# מערכת הלמידה שלי — Learning OS

מקום אחד שמרכז למידה מקצועית מתמשכת ב-AI, Product, R&D/Eng-Ops ו-Project/Strategy.
ארבעה טאבים: **Digest** (סיכומי עברית עם משמעויות + המלצות פרקטיות + לינקים; כולל פוסטים
שמורים מלינקדאין, מתוארכים לפי תאריך הפרסום האמיתי), **בייסליין** (צ'קליסט בסט-פרקטיסז עם
"איך עושים"), **מקורות** (ספרייה עם "כדאי במייל"), ו**ארכיון** (פריטים שהתיישנו מה-Digest
מעל 14 יום — שום דבר לא נעלם).

## 🌐 גישה והרצה — כולל במכונה / יוזר חדשים

**לצפייה — אפס הגדרה:** פתח בכל דפדפן, מכל מכונה → **https://zeev-l.github.io/learning-os/**

**לרענון (למשוך מקורות חדשים ולעדכן) — הגדרה חד-פעמית:**
1. `git clone https://github.com/Zeev-L/learning-os.git`
2. `cd learning-os && npm install`  (תלויות מנוע ה-RSS)
3. `gh auth login`  (כדי לדחוף עדכונים חזרה ל-repo)
4. בשיחת Claude Code בתיקייה, בקש: **"רענן את ה-Digest"**. לעדכון הגרסה החיה: `git add -A && git commit -m refresh && git push`.

**המערכת RSS-only ונייד לחלוטין** — אין תלות ב-Gmail או ביוזר אישי. כל המקורות (`data/sources.js`),
הבייסליין וההיסטוריה נוסעים עם ה-clone, אז זה עובד זהה בכל מכונה/יוזר. מדריך מלא: [SETUP.md](SETUP.md).

## שימוש יומיומי

- **לפתוח את הדשבורד:** לחיצה כפולה על `index.html` (נפתח בדפדפן). לא צריך שרת.
- **לרענן את ה-Digest:** "דרך Claude" — בשיחת Claude Code לבקש "רענן את ה-Digest". הרענון מושך **שני מקורות** ומסכם לעברית: **(1) RSS** מהמקורות ב-`sources.js`, ו**(2) GitHub Trending** — 3 הכלים החמים הרלוונטיים לתחומים/לעבודה שלך (מסונן, לא raw). המערכת **RSS-only ונייד לחלוטין** — בלי תלות ב-Gmail או ביוזר אישי. לחלופין `עדכן-digest.command` מריץ את מנוע ה-RSS.
- **רענון אוטומטי שבועי (אופציונלי, בלי מפתח API):** `bash scripts/install-schedule.sh` מתזמן ריצה כל יום ראשון 08:00 (launchd) — `claude -p` headless מרענן, מסכם לעברית, ודוחף לבד. **בדיקה ידנית קודם:** `bash scripts/auto-refresh.sh` (צפה ב-`auto-refresh.log`). להסרה: `launchctl unload ~/Library/LaunchAgents/com.zeev.learningos.plist`. דורש: המחשב דלוק בזמן הריצה.
- **הקראה קולית:** הפקד הצף למטה — `▶ הקראה` מקריא את הטאב הנוכחי כרטיס-אחר-כרטיס (בעברית, קול Carmit), עם השהה/עצור ומהירות (1× / 1.25× / 1.5×). **לבחור טקסט בכרטיס → `▶ הקרא מכאן`** מתחיל מאותו פריט.

## תחזוקת מקורות (קצב)
- **מחקר עומק מלא — כל רבעון (~90 יום):** הוספה/הסרה של מקורות + אימות feeds.
- **בדיקת בריאות feeds — חודשי:** `node scripts/check-feeds.mjs` (מסמן RSS מתים/תקועים).
- **טריגר אוטומטי:** בכל "רענן את ה-Digest", Claude בודק אם עברו >90 יום מ-`sources.updatedAt` ומזכיר על אודיט. פירוט: [MAINTENANCE.md](MAINTENANCE.md).

## מבנה

| קובץ | תפקיד |
|---|---|
| `index.html` | הדשבורד (RTL עברית, בייסליין + Digest + מקורות). קורא את `data/*.js`. |
| `data/baseline.js` | צ'קליסט הבסט-פרקטיסז. הסימונים נשמרים בדפדפן (localStorage). |
| `data/sources.js` | ספריית המקורות. **כאן מוסיפים/מסירים מקורות ידנית.** |
| `data/digest.js` | סיכומי התוכן. פריטים עם `manual:true` נכתבו ידנית באיכות גבוהה ו**נשמרים בכל ריצת סקריפט**. |
| `scripts/fetch-digest.mjs` | מנוע ה-RSS: מושך את התוכן המלא, ממזג (dedup), ומסכם. |

## הוספת מקור

לערוך את `data/sources.js` ולהוסיף שורה:
```js
{ domain:"ai", tier:"start", name:"שם", author:"מחבר", platform:"Substack",
  url:"https://...", rss:"https://.../feed", why:"למה שווה" }
```
`domain`: `ai|pm|rd|st` · `tier`: `start|deep` · `region:"il"` למקור ישראלי · `rss:false` אם אין feed.

## תקצירי עברית אוטומטיים לפריטים חדשים

הפריטים הידניים (`manual:true`) כבר מסוכמים באיכות גבוהה בעברית עם משמעויות והמלצות.
**פריטים חדשים שנמשכים מ-RSS** מסוכמים כך:
- **בלי מפתח:** תקציר נקי מהתוכן המלא של המאמר (לרוב בשפת המקור, בלי סעיף משמעויות/המלצות).
- **עם `ANTHROPIC_API_KEY`:** תקציר עברי מלא + משמעויות + המלצות פרקטיות (כמו הפריטים הידניים).
  1. `cp .env.example .env`  2. למלא את המפתח (לא לעשות commit — כבר ב-`.gitignore`).

> אין `claude` CLI מותקן, ולכן לתקצור-עברית אוטומטי נדרש מפתח API (זול עם `claude-sonnet-5`).
> לחלופין: לבקש מ-Claude Code לרענן את ה-Digest (משתמש במנוי הקיים, בלי מפתח).

## תזמון אוטומטי (אופציונלי)

ריצה שבועית אוטומטית (ראשון 08:00) דרך launchd:
```bash
cp com.zeev.learningos.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.zeev.learningos.plist
```
(המחשב צריך להיות דלוק. הנתיבים ב-plist מותאמים לתיקייה זו — לעדכן אם מזיזים.)
